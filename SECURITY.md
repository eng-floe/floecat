# Security Policy

## Reporting a Vulnerability

If you discover a potential security vulnerability in Floecat, do **not** open a public issue or pull request.

Please report it privately to the maintainers:

- **Contact:** [floecat-maintainers@floedb.ai](mailto:floecat-maintainers@floedb.ai)

Include as much detail as possible to help us reproduce and triage quickly:

- Affected component(s) and version/commit
- Description of the issue and potential impact
- Reproduction steps or proof of concept
- Any suggested mitigations or fixes

## Disclosure Expectations

- Please allow maintainers time to investigate and remediate before public disclosure.
- We will coordinate responsible disclosure timing after triage.
- We may request additional validation details while investigating.

## What to Expect After Reporting

- We will acknowledge receipt as quickly as possible.
- We will assess severity and impact, then prioritize remediation.
- We will communicate status updates and coordinate publication of any advisory when appropriate.

## Scope

This policy covers security vulnerabilities in this repository and its maintained components.

For non-security bugs or feature requests, use the normal issue and PR workflow.
